# 데이터로 진단하는 업무지구의 성공과 실패 — 판교테크노밸리 비교분석 시스템

가천대학교 스마트시티학과 기말과제 | 비교 대상: **판교 제1테크노밸리**(성공) vs **송도국제업무지구(IBD)**(저조)

🔗 **저장소:** https://github.com/jingij2-cmd/pangyo-songdo-analysis
🔗 **배포 시스템(GitHub Pages):** https://jingij2-cmd.github.io/pangyo-songdo-analysis/

## 1. 개요
수도권 대표 성공 업무지구(판교 제1테크노밸리)와 활성화가 저조한 업무단지(송도 IBD)를
공공데이터로 비교 분석한다. 핵심 질문은 **"왜 한 곳은 성공하고 한 곳은 실패했는가"**이며,
대중교통 접근성·토지이용·업무시장 지표를 동일 기준으로 정량 비교한다.

## 2. 분석 범위
- **공간범위**: 본인이 정의한 구역계 — 판교 제1테크노밸리(약 0.81㎢), 송도 IBD(약 5.77㎢)
- **시간범위**: 인구격자 2024, 용도지역 2023(판교)/2008(송도) 고시, 공실률 2024 3분기~2026 1분기
- **공간단위**: 100m 인구격자 / 용도지역 폴리곤 / 지하철 역(station) / 집계구
- **운영망 기준일**: 2026-06-18

## 4. 분석 방법 (`/code`)
- `dijkstra.py` — 핵심역(판교역·국제업무지구역) 기준 지하철 최단시간 탐색(환승 대기 포함)
- `reachpop.py` — 등시간권 × 100m 인구격자 결합 → 30·60분 도달가능 인구 (보행 1.2m/s·도로우회 1.3배)
- `landuse.py` — VWorld 용도지역을 구역경계로 클립 → 용도지역 구성비·혼합도(엔트로피 LUM)
- `build_map.py` — 위 결과를 웹지도(index.html)용 GeoJSON으로 변환
- 좌표계: EPSG:5179(UTM-K)·5174(중부원점) ↔ WGS84 변환을 직접 구현

## 5. 주요 결과
| 지표 | 판교 제1TV | 송도 IBD |
|---|---|---|
| 30분 도달가능 인구 | 약 105만명 | 약 31만명 |
| 60분 도달가능 인구 | 약 788만명 | 약 250만명 |
| 30분 도달역 수 | 80개 | 24개 |
| 용도지역 혼합도 LUM | 0.27 (준주거 집중) | 0.79 (분산·녹지과다) |
| 오피스 공실률(권역) | 분당역세권 14.3% | 인천 16.9% (전국 8.8%) |

판교는 신분당선·경강선 환승으로 서울 노동시장에 직결되어 30분 catchment가 송도의 **3.4배**.
송도는 인천1호선 단일 의존으로 도달범위가 인천 내부에 갇히고, 토지이용도 주거·녹지 위주로 분산되어
업무 핵심기능이 미성숙하다.
